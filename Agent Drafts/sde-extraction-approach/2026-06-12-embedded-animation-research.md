# Embedded organic data-animation — research brief

Research for three visuals: **(1)** "watch the fingerprint form" (a SHA-256 assembling from
the user's own extracted quotes), **(2)** a birds-eye relational scan over the page, **(3)**
a live telemetry / validation-gate pipeline diagram.

## Recommended technical stack (fits Vite + React 19 + TS + Tailwind v4)

Standardize on **four** libraries; SVG as the default substrate throughout (crisp text/lines
over a document, DOM-native, accessible). Canvas/WebGL held in reserve behind node-count
thresholds.

| Library | Role | Why | React 19 |
|---|---|---|---|
| **GSAP 3 + `@gsap/react`** | orchestration for visuals 1 & 3 | only engine here with frame-accurate `timeline.seek()` → true scrub/step; now fully free (Webflow, Apr 2025) | ✅ `useGSAP()` |
| **@xyflow/react (React Flow v12)** | the telemetry pipeline (visual 3) | purpose-built node+edge engine, animated edges, custom health nodes from live state; MIT | ✅ native |
| **d3-force → own SVG** | relational overlay (visual 2) | the physics under every force-graph; render as SVG so nodes compose with the document overlay + stay accessible | ✅ (substrate-agnostic) |
| **react-pdf + pdf.js text layer** | document render + quote highlight (already a dep, v10.4.1) | per-glyph spans to anchor highlights/connectors | ✅ |

Connectors are **not** a library: hand-rolled SVG overlay driven by `getBoundingClientRect()`,
optionally **perfect-arrows** (~2 KB) for curve math. (The maintained arrow libs are dead:
leader-line archived Apr 2025, react-xarrows unmaintained since 2021.) Net new weight ≈ 200 KB.

**Substrate switch points:** SVG is fine to ~1–2k animated nodes; past ~2k switch visual 2 to
Canvas2D (keep d3-force); WebGL (PixiJS/sigma.js) only past ~50–100k — not a realistic page.

**Do NOT** add a second animation engine (don't mix Framer Motion springs with GSAP on the same
elements), and **don't** reach for Lottie; if pre-authored data-driven motion is ever wanted, use
Rive (state machines + data binding), not Lottie.

## Design principles (organic AND honest)

From The Pudding, Distill, Emil Kowalski, Rauno Freiberg, Sigstore/C2PA, Smashing's AI-transparency
patterns, Prodigy/Label Studio, Obsidian:

1. **Reader sets the pace; motion answers intent** — scroll/hover/click-triggered reveals
   (Scrollama + IntersectionObserver), never autoplay theater.
2. **Anchor every animation to something real** — token offsets (Prodigy), page bounding boxes
   (Layout Parser/DocLayNet), the actual input bytes (hash visualizers), real DOM position.
3. **Restraint = nativeness** — sub-300ms, ease-out, transform/opacity only, 60fps, interruptible,
   `prefers-reduced-motion` honored, stagger settling for physicality. Showpiece moments fire
   **once / on-demand**, not every interaction (a researcher seeing it 50×/day will hate a 2s play).
4. **Trust comes from re-derivability, not badges** — Sigstore/C2PA: let the user re-hash/inspect/
   click-back-to-source. Integrity demonstrated by fragility (the hash breaks if tampered) beats a
   green checkmark.
5. **Scope relational views locally** — Obsidian's lesson: global force-graph = hairball (~500 nodes);
   local graph (current datum + its neighborhood) = useful. Build visual 2 as a *local* x-ray.
6. **Label the epistemic status; don't fake the process** — call a replay a replay (Smashing "Audit
   Trail" = "replay the decision logic"); let interpretive layers look interpretive (Excalidraw's
   sketch aesthetic); keep the literal substrate (real offsets, real hash) literal. "Colleagues,
   not magic tricks."

## Concrete "do this"

**Fingerprint forms (visual 1):** feed it the user's *real* extracted quotes, visibly flowing in
(like the SHA-256 byte→schedule→rounds→digest visualizers: github.com/in3rsha/sha256-animation,
github.com/bitcoin-dev-project/hashes-visualizer); stagger their arrival; settle into the final
digest. Offer a quiet "verify this yourself" (re-hash) affordance; optionally show the avalanche
effect (change a quote → hash changes) to convert the hash from mysticism to mechanism. Fire once
on first formation, instant/static thereafter, reduced-motion crossfade fallback.

**Birds-eye relational scan (visual 2):** labeled-overlay over the real page (document-layout-analysis
pattern: Layout Parser, DocLayNet — boxes registered to page geometry, color-coded by type) = the
honest substrate; draw the *relational* layer (datum ↔ supporting text ↔ context, à la scite.ai
citation context) **only for the focused datum** (Obsidian local-graph) to avoid the hairball. Use
the span→chip link (Prodigy/Label Studio: highlight in place + adjacent value chip, color = relation
type) rather than free-floating leader lines. Give it a deliberately interpretive visual treatment +
a one-line frame ("an interpretive view of how this datum relates to its context — not a literal
model trace").

**Scan/reveal (visual 3 of the brief / threads through all):** Pudding `position: sticky` — pin the
source page, accrete highlights/relationships step-by-step as the reader scrolls, each step tied to a
real reading position via IntersectionObserver.
